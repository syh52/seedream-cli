# SeeDream UI Refactor Design

> 参考 Midjourney 移动端（/explore, /imagine, /organize），对 SeeDream 进行全面 UI/UX 重构。

## 设计决策摘要

| 决策项 | 结论 |
|--------|------|
| 目标平台 | 移动优先（保持现有方向） |
| Tab 结构 | 2 Tab：Gallery + Create |
| Gallery 筛选 | All / Liked（去掉 Mine） |
| Create 布局 | Midjourney 风格卡片流（prompt + 标签 + 2x2 网格） |
| 图片详情 | 全屏详情页 + Bottom Sheet（替代 Dialog 弹窗） |
| 全局输入栏 | 每个页面底部都有（Gallery + Create） |
| Remix 逻辑 | 回填完整上下文：prompt + 参考图 + 尺寸 + strength |

---

## 1. 整体架构

### 页面结构

```
┌─────────────────────────────────┐
│         SeeDream                │
├─────────────────────────────────┤
│                                 │
│   Gallery 页 (/)                │
│   或                            │
│   Create 页 (/create)           │
│                                 │
├─────────────────────────────────┤
│  [🖼+] prompt...          [⚙]  │ ← 全局输入栏（每页都有）
│  Gallery          Create        │ ← 底部 TabBar（2 tab）
└─────────────────────────────────┘
```

### 三个核心变化

**1. 全局 Prompt 输入栏**
- 从 Create 页面专属 → 改为每个页面底部都有
- 固定在 TabBar 上方，用户在任何页面都能随时输入 prompt 开始创作
- 点击输入后展开完整表单（尺寸选择、strength 滑块）
- 提交后自动跳转到 Create 页面查看进度

**2. Gallery 筛选简化**
- 从 `All / Mine / Liked` → 简化为 `All / Liked`
- 保持 2 列 masonry 瀑布流
- 图片卡片无 overlay，点击进入全屏详情页

**3. Create 页面重做**
- 从 ColorStrip + 水平滚动 → 改为 Midjourney 风格卡片流
- 每个 Entry = prompt 文字 + 参数标签 + 2x2 图片网格
- 纵向无限滚动，按时间倒序排列

## 2. Create 页面详细设计

### 页面布局

```
┌─────────────────────────────────┐
│ Create                      🔍  │ ← 固定顶栏
├─────────────────────────────────┤
│                                 │
│ ┌─────────────────────────────┐ │
│ │ 🔴 Imagine       🗑 📋 ↻   │ │ ← Entry 头部
│ │ a cute cat sitting on...    │ │ ← prompt（截断 2 行）
│ │ [3:4] [image mode]          │ │ ← 参数 pill badges
│ │ ┌──────────┬──────────┐     │ │
│ │ │ ██████░░ │ ██████░░ │     │ │ ← 2x2 网格
│ │ ├──────────┼──────────┤     │ │    pending = 脉冲骨架
│ │ │ ████████ │ ██░░░░░░ │     │ │    done = 图片淡入
│ │ └──────────┴──────────┘     │ │
│ └─────────────────────────────┘ │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ 🟢 Imagine       🗑 📋 ↻   │ │ ← 已完成的 Entry
│ │ minimalist bathroom,...     │ │
│ │ [16:9] [text mode]          │ │
│ │ ┌──────────┬──────────┐     │ │
│ │ │   img1   │   img2   │     │ │
│ │ ├──────────┼──────────┤     │ │
│ │ │   img3   │   img4   │     │ │
│ │ └──────────┴──────────┘     │ │
│ └─────────────────────────────┘ │
│                                 │
│ (更多 entries...)               │
│                                 │
├─────────────────────────────────┤
│  [🖼+] prompt...          [⚙]  │
│  Gallery          Create        │
└─────────────────────────────────┘
```

### EntryCard 组件

**头部行**：

| 元素 | 说明 |
|------|------|
| 状态指示 | 🔴 active（生成中）/ 🟢 done / 🔴 failed |
| `Imagine` 标签 | 红色/emerald 文字，表示生成来源 |
| `🗑` 删除 | 删除整个 Entry（软删除） |
| `📋` 复制 | 复制 prompt 文字 |
| `↻` Remix | 将 prompt + 参考图 + 尺寸 + strength 回填到输入栏 |

**Prompt 区域**：
- 最多显示 2 行，超出截断 `...`
- 字体：正文字号，`text-zinc-300`

**参数标签（Pill Badges）**：
- 水平排列，超出换行
- 显示内容：尺寸（`3:4`）、模式（`text` / `image` / `multi`）、strength（如 `str 0.5`，仅 image/multi 模式显示）
- 样式：`bg-zinc-800 text-zinc-400 rounded-full px-2 py-0.5 text-xs`

**2x2 图片网格**：
- 固定 2 列，图片保持原始比例
- 3 种状态：
  - `pending`：脉冲骨架屏（`animate-pulse bg-zinc-800`）
  - `done`：图片淡入（`animate-fadeIn`），点击进入详情
  - `failed`：灰色背景 + 红色 X 图标
- 图片间距：`gap-1`

## 3. 图片详情页

点击 Create 或 Gallery 中的任何图片，进入全屏详情页（替代当前 Dialog 弹窗）：

```
┌─────────────────────────────────┐
│                             ✕   │ ← 关闭按钮
│                                 │
│ ┌─┐ ┌─┐ ┌─┐ ┌─┐               │ ← 同批次缩略图条
│ └─┘ └─┘ └─┘ └─┘               │   （水平滚动，当前图高亮）
│                                 │
│         ┌───────────┐           │
│         │           │           │
│         │   大 图    │           │ ← 全宽展示，保持比例
│         │           │           │
│         └───────────┘           │
│                                 │
│ a cute cat sitting on a...      │ ← prompt 全文（可展开）
│ [3:4] [image mode] [str 0.5]   │ ← 参数标签
│                                 │
│    ♡         ⬇         ≡        │ ← 3 个操作按钮
└─────────────────────────────────┘
```

### 缩略图条
- 来自同一个 Entry 的所有图片（通常 4 张）
- 水平排列，当前图有高亮边框（`ring-2 ring-emerald-500`）
- 左右滑动切换图片

### 底部操作栏（3 按钮）

| 按钮 | 功能 | 交互 |
|------|------|------|
| `♡` | 收藏/取消 | 点击 toggle，已收藏时实心红心 |
| `⬇` | 下载 | 直接下载原图到设备 |
| `≡` | 更多 | 打开 Bottom Sheet 菜单 |

### Bottom Sheet 菜单

```
┌─────────────────────────────────┐
│ ┌────┐ a cute cat sitting...    │ ← 缩略图 + prompt 摘要
│ └────┘ by Yihang Shen           │
├─────────────────────────────────┤
│ 📋  Copy prompt                 │
│ ↻   Remix                       │ ← 回填 prompt + 参考图 + 尺寸 + strength
│ 🗑  Delete                      │
└─────────────────────────────────┘
```

### Remix 逻辑
1. 从 Entry 读取完整上下文：`prompt`、`referenceImageUrls`、`size`、`strength`、`mode`
2. 全部回填到全局输入栏
3. 关闭详情页，输入栏展开为编辑状态
4. 用户可修改后重新提交

## 5. 全局输入栏

固定在 TabBar 上方，Gallery 和 Create 页面都有。

### 收起状态（默认）

```
┌─────────────────────────────────┐
│ [🖼+]  prompt...          [⚙]  │ ← 单行，点击展开
└─────────────────────────────────┘
```

### 展开状态（点击输入框或 Remix 触发）

```
┌─────────────────────────────────┐
│ ┌───────────────────────── ▲ ┐  │
│ │ a cute cat sitting on a    │  │ ← 多行输入框（自动增长）
│ │ windowsill...               │  │
│ └────────────────────────────┘  │
│ [3:4 ˅]  [str 0.5 ──●──]  [➤] │ ← 尺寸下拉 + strength + 提交
└─────────────────────────────────┘
```

### 关键行为
- `[🖼+]` 按钮：打开图片选择（相机/相册）
- `[⚙]` 按钮：展开选项行（尺寸 + strength）
- strength 滑块只在有参考图时显示
- 提交后：输入栏收起，如果不在 Create 页面则自动跳转
- `Enter` 提交，`Shift+Enter` 换行

## 6. 视觉设计风格

向 Midjourney 的「沉浸暗色」风格靠拢，让图片成为绝对主角。

| 属性 | 当前 | 重构后 |
|------|------|--------|
| 背景 | `zinc-950` | `#0d0e12`（更纯的深黑） |
| 字体 | Syne + Space Mono | 保持不变 |
| 卡片背景 | 各种 zinc-900/800 | `zinc-900/50`（半透明） |
| 强调色 | `emerald-500` | 保持 emerald |
| 图片间距 | `gap-3` | `gap-1`（紧凑） |
| 圆角 | 混合 | 统一 `rounded-lg`（8px） |
| 参数标签 | 无 | 新增 pill badges |
| 操作按钮 | 彩色图标 | 单色（`text-zinc-400`），active 时变亮 |

### 设计原则
- **图片至上**：减少所有非图片元素的视觉权重
- **紧凑间距**：`gap-1` 让图片紧密排列
- **信息克制**：默认只显示最少信息，详情收进子页面

## 8. 数据模型影响

现有 `Entry` 和 `EntryImage` 类型无需修改，完全兼容新 UI。

Remix 功能需要从 Entry 读取以下字段：
- `prompt` — 回填输入框
- `referenceImageUrls` — 回填参考图（image/multi 模式）
- `size` — 回填尺寸选择
- `strength` — 回填 strength 滑块
- `mode` — 决定输入栏的模式状态
